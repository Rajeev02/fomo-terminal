3. Issues & Resolutions

1. BirdEye API Reliability

Problem: API rate limits (HTTP 429) and intermittent latency.

Solution: Integrated automatic API key rotation and built mock data fallback engines to guarantee 100% uptime.

2. Theme Hydration

Problem: Next.js SSR clashing with Zustand localStorage client cache.

Solution: Deferred rendering of theme-dependent visual assets using a client-side mounted state in useEffect.

3. Privy Solana Wallet Filtering

Problem: Privy returned EVM wallets first in multi-chain browser environments.

Solution: Refactored auth hooks to strictly query for wallets matching chainType === "solana".

4. Responsive Mobile Layout

Problem: Vertical stacking of three-column terminal layout cluttered mobile viewports.

Solution: Designed a mobile bottom navigation bar and toggle visibility CSS classes to show a single focused pane.

4. Proposed Improvements

  • 1. Multi-RPC Failover: Alternate connection links (Helius, QuickNode, Alchemy) for resilient txn routing.
  • 2. Portfolio Analytics: Add charts tracking historical asset P/L and detailed holdings.
  • 3. Database Persistence: Enable persistent watchlists and custom price alerts using Supabase.
  • 4. Fiat On-Ramp: Link Privy with Stripe or MoonPay to support direct credit card purchases.
  • 5. Automated Testing: Introduce Jest unit tests and Playwright E2E testing for critical swap routes.
  • 6. Redis Caching: Cache responses on BirdEye trending lists to minimize API bounds.
